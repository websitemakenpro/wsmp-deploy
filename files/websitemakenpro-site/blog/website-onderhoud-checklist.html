Website onderhoud: de checklist voor elk kwartaal in 2026.

Een website is geen statisch eindproduct — hij moet bijgehouden worden. Deze checklist bevat de 18 concrete taken die MKB-eigenaren elk kwartaal moeten afvinken. Voor WordPress, custom en statische sites.

A
Agah
BSc Computer Engineer · WebsiteMakenPro
28 maart 2026·9 min leestijd

"We hebben een nieuwe website laten maken" is een uitspraak die ondernemers vaak doen. "We onderhouden onze website goed" is een uitspraak die ze zelden doen — en waar je eerlijkheid meestal snel verdwijnt als je doorvraagt.

De realiteit: een website is zoals een auto. Na aanschaf moet je onderhoud plegen, anders gaat de kwaliteit achteruit. Met auto's is dat iedereen duidelijk. Met websites ervaren eigenaars het pas als er iets misgaat — een hack, een crash, Google rankings die stilletjes wegzakken.

Dit artikel geeft je een concrete kwartaal-checklist. 18 taken verdeeld over security, performance, content, en SEO. Voor WordPress-sites én voor moderne (static) sites. Reken ongeveer 2-4 uur per kwartaal voor een MKB-site.

Maandelijks — 5 minuten werk

Wat je elke maand moet doen, ook als je geen tijd hebt:

  1. Check of de site überhaupt werkt — klinkt raar, maar veel eigenaars hebben een half-jaar lang een kapotte site zonder het te weten. Uptime-monitoring tools (UptimeRobot gratis) sturen je een mail als de site offline is.
  2. Backup bevestigen — je hosting of CMS maakt backups, maar maakt het ook écht? Eens per maand een test-download doen is verstandig.
  3. Security updates installeren — WordPress-sites: plugin en theme updates. Uitstellen is vragen om problemen.

Kwartaal — security

4. Alle plugins en themes updaten (WordPress)

60% van WordPress-breaches komt via verouderde plugins. Update élk kwartaal, controleer compatibility eerst in staging als je kunt. Verwijder plugins die je niet meer gebruikt.

5. Admin-wachtwoorden veranderen

Elk kwartaal, met een wachtwoord-manager. Lijkt overdreven tot er iets gebeurt. Gebruik nooit hetzelfde wachtwoord als op andere services.

6. Two-factor authenticatie checken

Voor WordPress admin, hosting-panel, domain-registrar. Als 2FA niet actief is: direct aanzetten. Tools als Wordfence (WordPress) of Cloudflare Access doen dit eenvoudig.

7. Gebruikersrechten controleren

Wie heeft er nog toegang? Ex-medewerkers, oude bureau dat je ooit gebruikte? Review de user-lijst en verwijder wat niet meer nodig is.

8. SSL certificate check

Let's Encrypt vernieuwt automatisch (meestal), maar kan fout gaan. Browser-melding "niet beveiligd" = directe klantschrik. Controleer expiry-datum.

Kwartaal — performance

9. PageSpeed check uitvoeren

Run PageSpeed Insights op je homepage en 2-3 belangrijkste pagina's. Let op: Core Web Vitals scores. Zakken ze? Onderzoek waarom.

10. Broken links controleren

Tools zoals Ahrefs Site Audit (gratis beperkt) of Screaming Frog (gratis tot 500 URLs) vinden gebroken links. Gebroken links kosten ranking én irriteren bezoekers.

11. Image-optimization audit

Check of nieuwe content-uploads images correct geoptimaliseerd zijn. Gebruikers uploaden vaak onbewerkte iPhone-foto's van 5MB+ in je CMS.

12. Mobile-experience test

Test je site op daadwerkelijke telefoons — niet alleen Chrome DevTools mobile-emulatie. Verschillende schermgroottes, verschillende browsers (Safari iOS, Chrome Android).

Kwartaal — content & SEO

13. Google Search Console check

Zoek naar: daling in impressies, stijging in CTR, coverage-errors, sitemap-issues. 15 minuten lezen per kwartaal voorkomt dat problemen maanden sluimeren.

14. Top pagina's reviewen

Welke 5 pagina's krijgen het meeste verkeer? Zijn de teksten up-to-date? Kloppen prijzen, contactgegevens, product-informatie? Oude content die niet meer klopt kost je klanten.

15. Contactformulieren en CTA's testen

Verstuur een test-bericht via elk formulier. Kom je e-mails aan? Werkt de WhatsApp-link? Een kapot contactformulier kan maanden blijven zitten zonder dat iemand het merkt — en miljoenen euros aan verloren leads opleveren.

16. Analytics doorlopen

Google Analytics 4: is er een maand-over-maand dip? Groei? Waar komen bezoekers vandaan? Dit hoeft geen diepe SEO-analyse — een half uur scrollen en noteren wat opvalt.

17. Legal pages review

Privacybeleid, algemene voorwaarden, cookie-banner. AVG-wet verandert, je bedrijf verandert. Check elk kwartaal of deze nog kloppen met je huidige praktijk.

18. Strategische review

Laatste taak: zit de site er nog bij zoals je wilt? Zijn er features die gebruikt worden? Zijn er klacht-patronen in klant-mails? Dit is 30 minuten denken, geen doen — maar het waardevolste onderdeel van onderhoud.

Verschil tussen WordPress en statische sites

WordPress

2-4 uur per kwartaal

  • Plugin-updates verplicht
  • Security-monitoring kritiek
  • Database-optimalisatie nodig
  • Backup-workflow actief
Static / Jamstack

30-60 min per kwartaal

  • Geen plugin-onderhoud
  • Security door design hoger
  • Geen database om te beheren
  • Content-updates via Git

Een belangrijk gevolg van deze vergelijking: de keuze tussen WordPress en modern static heeft grote impact op hoeveel tijd en geld je aan onderhoud kwijt bent. Over 5 jaar gemeten is het verschil vaak €2.000-€5.000.

Onderhoud is geen creatief werk — het is discipline. Drie uur per kwartaal is niks, maar ongedaan laten leidt gegarandeerd tot problemen die veel duurder zijn.

Wat je uit kunt besteden

Niet alles hoeft door jou gedaan. Goede onderhoudscontracten (€29-€79/mnd) dekken typisch:

  • Updates (WordPress core, plugins, thema's)
  • Security monitoring en backups
  • Uptime monitoring
  • Maandelijkse performance-rapportage
  • Kleine content-updates (binnen limiet)

Wat je zelf moet blijven doen: strategische review, content-beslissingen, priority-setting. Die kennis kan een onderhoudsbureau niet hebben — alleen jij weet wat je bedrijf nodig heeft.

Kwartaal start-tip

Zet elke kwartaal-start (1 jan, 1 apr, 1 jul, 1 okt) een recurring calendar-afspraak van 2 uur in je agenda. "Website onderhoud". Doe dan deze checklist. Klaar. Geen uitstel mogelijk als het in je agenda staat.
